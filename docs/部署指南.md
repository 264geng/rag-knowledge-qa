## 方案一：Vercel + Railway 部署

### 第一步：部署后端到 Railway

#### 1.1 创建项目
1. 访问 https://railway.app
2. 用 GitHub 账号登录
3. 点击 "New Project" → "Deploy from GitHub repo"
4. 选择 `rag-knowledge-qa` 仓库

#### 1.2 配置环境变量
点击项目 → "Variables" → 添加以下变量：

```
OPENAI_API_KEY=sk-你的DeepSeek密钥
OPENAI_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
EMBEDDING_API_KEY=你的嵌入模型密钥
EMBEDDING_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
EMBEDDING_MODEL=doubao-embedding-vision-251215
EMBEDDING_PROVIDER=volcengine
JWT_SECRET_KEY=your-random-secret-key
```

#### 1.3 配置启动命令
点击 "Settings" → "Start Command"：
```
cd backend && uvicorn app.main:app --host 0.0.0.0 --port $PORT
```

#### 1.4 获取后端链接
1. 点击 "Settings" → "Networking"
2. 点击 "Generate Domain"
3. 记录链接，如：`https://rag-backend.up.railway.app`

#### 1.5 运行初始化脚本（重要！）

**在 Railway 部署成功后**，需要运行初始化脚本来创建测试数据：

**方法一：本地运行（推荐）**

```bash
# 设置环境变量指向你的云平台后端
set API_BASE_URL=https://rag-backend.up.railway.app/api

# 运行初始化脚本
cd backend
python init_data.py
```

**方法二：在 Railway 控制台运行**

1. 进入 Railway 项目
2. 点击后端服务
3. 点击 "Settings" → "Deploy" → "Custom Start Command"
4. 临时修改为：`cd backend && python init_data.py && uvicorn app.main:app --host 0.0.0.0 --port $PORT`
5. 触发重新部署
6. 部署成功后，改回原来的启动命令

---

### 第二步：部署前端到 Vercel

#### 2.1 导入项目
1. 访问 https://vercel.com
2. 点击 "Add New..." → "Project"
3. 选择 `rag-knowledge-qa` 仓库
4. 点击 "Import"

#### 2.2 配置项目
| 配置项 | 值 |
|--------|-----|
| Framework Preset | Vite |
| Root Directory | `frontend` |
| Build Command | `npm run build` |
| Output Directory | `dist` |

#### 2.3 添加环境变量
```
VITE_API_BASE_URL=https://rag-backend.up.railway.app/api
```

#### 2.4 部署
点击 "Deploy"，等待完成。

#### 2.5 获取前端链接
部署成功后会显示链接，如：`https://rag-knowledge-qa.vercel.app`

## 方案二：Render 部署（更简单）

### 第一步：部署后端

1. 访问 https://render.com
2. 点击 "New" → "Web Service"
3. 连接 GitHub 仓库
4. 配置：
   - Name: `rag-backend`
   - Runtime: `Python 3`
   - Build Command: `cd backend && pip install -r requirements.txt`
   - Start Command: `cd backend && uvicorn app.main:app --host 0.0.0.0 --port $PORT`
5. 添加环境变量（同方案一）
6. 点击 "Create Web Service"

### 第二步：部署前端

1. 点击 "New" → "Static Site"
2. 连接同一仓库
3. 配置：
   - Name: `rag-frontend`
   - Build Command: `cd frontend && npm install && npm run build`
   - Publish Directory: `frontend/dist`
4. 添加环境变量：
   ```
   VITE_API_BASE_URL=https://rag-backend.onrender.com/api
   ```
5. 点击 "Create Static Site"

### 第三步：运行初始化脚本

同方案一，在本地运行：

```bash
set API_BASE_URL=https://rag-backend.onrender.com/api
cd backend
python init_data.py
```

## 常见问题

### Q1: 部署后看到空白页面？
**原因**：没有运行初始化脚本
**解决**：运行 `python init_data.py`

### Q2: 智能问答没有回答？
**原因**：API密钥配置错误
**解决**：检查环境变量中的 OPENAI_API_KEY

### Q3: 前端无法访问后端？
**原因**：CORS 或环境变量配置错误
**解决**：检查 VITE_API_BASE_URL 是否正确

### Q4: 免费额度用完了？
**解决**：
- Railway 每月 500 小时免费
- Vercel 个人项目完全免费
- 可以考虑 Render、Fly.io 等替代方案
